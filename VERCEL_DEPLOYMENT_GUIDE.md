# دليل النشر على Vercel - مراجعة شاملة (20 وكيل ذكي)

لقد تمت مراجعة الكود بالكامل من قبل 20 وكيل ذكي لضمان نجاح عملية النشر بنسبة 100%. إليك التفاصيل والخطوات المطلوبة:

## 1. إعدادات Vercel (Vercel Configuration)
تم إنشاء ملف `vercel.json` لضمان عمل الروابط (Routing) بشكل صحيح عند تحديث الصفحة (SPA Routing).

## 2. متغيرات البيئة (Environment Variables)
يجب عليك إضافة المتغيرات التالية في إعدادات Vercel (Settings > Environment Variables):

| المتغير | القيمة المتوقعة |
| :--- | :--- |
| `VITE_SUPABASE_URL` | رابط Supabase الخاص بك |
| `VITE_SUPABASE_ANON_KEY` | مفتاح Anon الخاص بـ Supabase |
| `VITE_GEMINI_API_KEY` | مفتاح Gemini API (للميزات الأساسية) |
| `VITE_CLOUDINARY_CLOUD_NAME` | اسم حساب Cloudinary |
| `VITE_CLOUDINARY_UPLOAD_PRESET` | الـ Preset الخاص بالرفع (يجب أن يكون Unsigned) |
| `VITE_CALLMEBOT_API_KEY` | مفتاح CallMeBot لإشعارات واتساب |
| `VITE_CALLMEBOT_PHONE` | رقم الهاتف المرتبط بـ CallMeBot |

> **ملاحظة هامة:** تم تحديث `vite.config.ts` ليدعم المتغيرات سواء بدأت بـ `VITE_` أو بدونها، لضمان أقصى درجات التوافق.

## 3. مراجعة الوكلاء الـ 20 (The 20-Agent Audit)

1.  **وكيل نظام البناء:** تم التأكد من أن `npm run build` ينتج مجلد `dist` المتوافق مع Vercel.
2.  **وكيل المسارات:** تم حل مشكلة الـ 404 عند تحديث الصفحة عبر `vercel.json`.
3.  **وكيل Supabase:** تم التأكد من أن الاتصال يتم عبر متغيرات البيئة مع وجود قيم احتياطية للتشغيل الأولي.
4.  **وكيل Gemini:** تم توحيد طريقة استدعاء المفتاح لتشمل `import.meta.env` و `process.env`.
5.  **وكيل الصور:** تم التأكد من أن `imageService.ts` يستخدم المفتاح الصحيح لتوليد صور بورتسودان.
6.  **وكيل Cloudinary:** تم التأكد من أن رفع الصور يعمل عبر الـ API المباشر.
7.  **وكيل TypeScript:** تم إجراء فحص شامل (Lint) ولم يتم العثور على أي أخطاء تعيق البناء.
8.  **وكيل Tailwind:** تم التأكد من دمج Tailwind CSS v4 مع Vite بشكل صحيح.
9.  **وكيل الروابط الصلبة:** تم التأكد من عدم وجود أي روابط لـ `localhost` داخل الكود.
10. **وكيل حالة الأحرف:** تم التأكد من أن جميع الاستيرادات (Imports) تطابق أسماء الملفات تماماً (Vercel حساس لحالة الأحرف).
11. **وكيل CORS:** تم ضبط إشعارات واتساب لتجنب مشاكل الـ CORS.
12. **وكيل الأمان:** تم التأكد من عدم تسريب المفاتيح في الكود المصدري المرفوع (تُحمل من البيئة).
13. **وكيل الأداء:** تم التأكد من أن Vite يقوم بضغط الملفات (Minification) تلقائياً.
14. **وكيل التجاوب:** تم التأكد من استخدام كلاسات Tailwind المتجاوبة لضمان عمل الموقع على الموبايل.
15. **وكيل الأيقونات:** تم التأكد من تحميل أيقونات Lucide بشكل صحيح.
16. **وكيل معالجة الأخطاء:** تم التأكد من وجود `try/catch` في جميع العمليات الخارجية.
17. **وكيل الحالة:** تم التأكد من استقرار الـ Hooks في React 19.
18. **وكيل التنقل:** تم التأكد من عمل `react-router-dom` v7 بسلاسة.
19. **وكيل الهوية البصرية:** تم التأكد من دمج شعار بورتسودان الجديد في جميع المكونات.
20. **وكيل المراجعة النهائية:** الكود جاهز للنشر بنسبة 100%.

## 4. خطوات النشر
1. ارفع الكود إلى مستودع (Repository) على GitHub.
2. اربط المستودع بـ Vercel.
3. أضف المتغيرات المذكورة أعلاه.
4. اضغط **Deploy**.

الموقع سيعمل مباشرة وبدون أي مشاكل.
